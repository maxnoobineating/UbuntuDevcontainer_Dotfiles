If these are the only errors and it doesn't stop because of exceeding the error limit, that's really good enough. I am in a similar situation, but with plain clangd and a different editor. All of the include paths and necessary stuff is in the compile commands, and once the editor runs clangd it should be able to figure it out.

Run clangd with the args --limit-results=0 --all-scopes-completion --limit-references=0 --background-index --background-index-priority=background --enable-config for more thorough suggestions, then you can try silencing unimportant errors with ~/.config/clangd/config.yaml:

CompileFlags:
    Add:
        [
            -ferror-limit=99999,
            # -I,
            # /usr/lib/modules/6.18.2-arch2-1/build/include/,
            -Wno-implicit-function-declaration,
        ]
    # Remove:
    #     - -mlongcalls
    #     - -mdisable-hardware-atomics
    #     - -mno-target-align
    #     - -fno-shrink-wrap
    #     - -fstrict-volatile-bitfields
    #     - -fno-tree-switch-conversion
    #     - -fzero-init-padding-bits=all
    #     - -fno-malloc-dce
Diagnostics:
    Suppress:
        #         - pp_file_not_found
        - unused-includes
